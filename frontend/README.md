# Дашборд результатов исследования

Статический сайт с результатами магистерской ВКР на тему
«Исследование и оптимизация производительности Node.js-приложений: сравнительный
анализ стратегий кэширования и индексирования».

**Демо:** https://inthetouch.github.io/nodejs-mongodb-project/

---

## Что это

Дашборд показывает результаты 22 нагрузочных экспериментов на тестовом стенде
Node.js + MongoDB + Redis. Содержит:

- **Обзор** — тема работы и ключевые ссылки
- **Эксперименты** — таблица всех 22 прогонов с фильтрами
- **Графики** — 10 интерактивных диаграмм Chart.js с описанием и выводом для каждой
- **Защита от лавины** — отдельный раздел про single-flight
- **Параметры эксперимента** — характеристики стенда, среды и матрицы прогонов

## Технологии

- **HTML5 + CSS Grid** — без фреймворков и сборки, чистая статика
- **Vanilla JavaScript** — модули `app.js` (навигация, фильтры, таблица) и `charts.js` (Chart.js-графики)
- **Chart.js 4.4** — графики в академическом стиле matplotlib с подписями значений над столбцами
- **CSS-переменные** — две темы (светлая/тёмная) с автоматической перерисовкой графиков
  при переключении через `MutationObserver`

## Структура

```
frontend/
├── index.html              # одна страница со всеми разделами (SPA-навигация без рутера)
├── README.md
├── assets/
│   ├── style.css           # вёрстка, темы, sticky footer
│   ├── app.js              # навигация, темы, таблица экспериментов, single-flight
│   └── charts.js           # 10 графиков Chart.js + плагин подписей значений
└── data/
    └── runs.json           # агрегированные данные 22 прогонов
```

## Локальный запуск

Поскольку сайт делает `fetch('data/runs.json')`, открывать `index.html` напрямую
из файловой системы нельзя — браузер заблокирует CORS. Нужен любой HTTP-сервер:

```bash
cd frontend
python3 -m http.server 8000
# открыть http://localhost:8000
```

Или через Node.js:

```bash
npx serve frontend -p 8000
```

## Деплой на GitHub Pages

Деплой автоматический через GitHub Actions при пуше в `main`. Конфигурация в
`.github/workflows/deploy-pages.yml` — собирается артефакт из папки `frontend/`
и публикуется на Pages.

**Триггер:** push в main с изменениями в `frontend/**` или в workflow-файле.

## Темы

Светлая (по умолчанию) и тёмная. Переключение — кнопка в правом углу шапки.
Выбор пользователя сохраняется в `localStorage`.

При смене темы:
- CSS-переменные `--bg-*`, `--text-*`, `--border-*` подменяются на корневом элементе
- `MutationObserver` в `charts.js` ловит изменение `data-theme` и автоматически
  перерисовывает все ранее построенные графики с обновлёнными цветами осей и текста

## Графики

Реализованы программно через Chart.js — без зависимости от готовых изображений.
Числа зашиты в `charts.js` напрямую из таблицы результатов ВКР. Если потребуется
обновить — править `charts.js` (соответствующая функция `drawFigX`).

Каждый график:
- Заголовок над канвасом (HTML, выбираемый, не часть Chart.js)
- Подписи значений над столбцами через кастомный плагин `valueLabelsPlugin`
- Палитра как в matplotlib-картинках исходной ВКР: бирюзовый `#0E7C7B`,
  серый `#8E8E8E`, бежевый `#D5CFC0`, малиновый `#C2185B`
- Логарифмическая шкала на графиках с большим разбросом значений (рис. 3.0, 3.8)
- Lazy-рендеринг через `IntersectionObserver` — график строится только когда
  карточка попадает в зону видимости

## Что использовалось при разработке

- Node.js + MongoDB + Redis на стенде для генерации данных
- Grafana k6 как нагрузочный генератор
- Python (`aggregate_runs.py`) для агрегации сырых `.summary.json` + `.stats.json`
  в единый `runs.json`
- Браузерные тесты вручную для проверки навигации, тем и адаптивной вёрстки
